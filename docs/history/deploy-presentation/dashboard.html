Pull-based: CI never reaches into the cluster. Argo only ever pulls from git and GHCR. The pipeline only commits the digest — it never runs an explicit argocd app sync (combining that with auto-sync is the documented race-condition footgun).

⚠ Setup Status — Action Required

GitHub App: NOT CONFIGURED

Auto-deploy is inactive. The update-k8cluster job checks APP_READY (both secrets present) and no-ops gracefully until setup is complete — by design, so the workflow never hard-fails before setup.

  1. Create a GitHub App with Repository permission Contents: Read & write (only), installed on only Coldaine/coldaine-k8cluster.
  2. Add two repo secrets to RobotOverview:
    GITOPS_APP_CLIENT_ID GITOPS_APP_KEY
    Client ID — the workflow uses create-github-app-token@v3, which deprecated the older app-id input in favor of client-id. Remove the old K8CLUSTER_REPO_TOKEN PAT.
  3. Allow the robot-overview-delivery-bot to push to coldaine-k8cluster main (branch-protection bypass / allowlist — direct push, no PR).

Once all three exist, the next merge auto-deploys with no human step.

≣ Monitoring Runbook — the full check list

1 · Did the build + write-back run? (GitHub Actions) 
# RobotOverview → Actions → "Build RobotOverview image"
# for the merge commit. The update-k8cluster job log
# shows the pinned digest (or a no-op notice if the
# GitHub App is not configured).
2 · Did the digest land in git? (GitOps commit) 
# In coldaine-k8cluster main, look for commit:
#   chore(robot-overview): deploy <sha> [skip ci]
#   author: robot-overview-delivery-bot
# Confirm apps/robot-overview/deployment.yaml image:
#   ghcr.io/coldaine/robot-overview@sha256:<digest>
3 · Did Argo sync? 
kubectl get application robot-overview -n argocd
# expect: Synced / Healthy
4 · Is the new pod running? 
kubectl -n robot-overview get pods
# expect: 1/1 Running
5 · Is the pod on the new digest? 
kubectl -n robot-overview get deploy robot-overview \
  -o jsonpath='{.spec.template.spec.containers[0].image}'
6 · Is it serving? 
# In-cluster:
http://robot-overview.robot-overview.svc.cluster.local:80/
# Publicly: the Cloudflare Tunnel hostname
# (routed in the Cloudflare dashboard to that service)
